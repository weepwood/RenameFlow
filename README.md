# RenameFlow

RenameFlow 是一个浏览器中的本地文件批量重命名工作台。它通过 File System Access API 读取用户主动授权的目录，在本地生成重命名预览、检查冲突，并在获得写入权限后执行修改。

> 文件内容与文件名默认不会上传到任何服务器。

## 已实现功能

- 递归扫描本地目录与文件元数据
- 前缀、后缀、查找替换、正则替换
- 连续空格/下划线标准化
- 自动编号、扩展名小写
- Windows 非法字符与保留名称检查
- 批次内重名、现有文件占用检测
- 原名称 / 新名称实时对照预览
- 两阶段临时名称执行，支持交换与连续重命名
- 原生 `move()` 优先，校验后的复制-删除降级
- 当前会话反向撤销
- 本地操作历史
- CSV、JSON、PowerShell 方案导出
- 无支持浏览器下的示例与规划模式

## 浏览器要求

建议使用最新版 Chromium 浏览器（Chrome 或 Edge）。目录选择与写入依赖 File System Access API。部分浏览器可能无法原生重命名，RenameFlow 会退回到“复制文件 → 校验大小 → 删除原文件”的兼容流程。

请先在测试目录中验证。重要文件应保留备份。

## 使用流程

1. 点击“选择文件夹”，授权读取目录。
2. 在右侧组合重命名规则。
3. 检查中间表格中的新名称与冲突提示。
4. 可先导出 CSV、JSON 或 PowerShell 方案。
5. 点击“执行”，此时才请求写入权限。
6. 操作完成后，可在当前页面尝试撤销。

## 安全边界

- 应用不会自动扫描磁盘，只能访问用户主动选择的目录。
- 首次选择目录只请求读取权限，执行时再请求读写权限。
- 撤销依赖当前会话中的文件句柄；关闭页面后不承诺可撤销。
- 复制-删除降级模式会校验文件大小，但不能替代完整备份。
- 操作历史只保存名称、时间与结果，不保存文件内容。
