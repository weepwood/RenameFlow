# 安全与隐私

## 本地优先

RenameFlow 没有后端接口，也不包含第三方分析脚本。目录内容、文件内容和文件名只在当前浏览器页面中处理。

## 权限最小化

- 用户点击“选择文件夹”后才调用目录选择器。
- 扫描阶段只申请读取权限。
- 用户明确点击“执行”后才申请读写权限。
- 浏览器可随时撤销权限，应用会把拒绝视为终止条件。

## 写入风险

兼容重命名可能使用复制-删除流程。实现会在删除源文件前比较目标文件大小，但断电、浏览器崩溃、磁盘故障和外部程序同时修改文件等情况仍可能导致异常。

建议：

1. 首次使用时选择临时测试目录。
2. 重要数据保留独立备份。
3. 大文件批量操作时避免关闭标签页。
4. 执行前导出 CSV 或 JSON 方案。

## 操作历史

历史记录保存在 `localStorage`，最多保留 20 条，只包含目录显示名称、文件名变化、时间和执行结果。不会保存文件内容、文件句柄或绝对路径。
